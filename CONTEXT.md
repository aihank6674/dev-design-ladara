# LADRA (Local Automated Document Redaction & Architecture Assessment System)

LADRA is an on-premise system that desensitizes sensitive enterprise architecture documents and uses local AI agents to perform architecture impact, DevOps, and infrastructure assessments.

## Language

**Raw Document**:
A confidential enterprise architecture or API document submitted by internal teams or external partners before desensitization.
_Avoid_: Source file, input doc, raw input

**Desensitized Artifact**:
A redacted version of a Raw Document (or accompanying change log) from which sensitive terms, IP addresses, secrets, and sensitive diagram regions have been masked.
_Avoid_: Clean file, redacted doc, output artifact

**Document Processing & Redaction Engine**:
The deterministic engine responsible for document diffing, text regex/lookup masking, and image OCR blackout before data reaches AI agents.
_Avoid_: Masker, sanitiser, redact pipeline

**Agent Harness**:
The Microsoft Agent Framework orchestration layer that executes specialized multi-agent roles using desensitized artifacts and local LLM endpoints.
_Avoid_: AI framework, agent runner, LLM wrapper

**Base Environment Context**:
The reference configuration files describing physical data center specs, network topology, and Kubernetes cluster baselines used by agents during assessment.
_Avoid_: Environment config, infra baseline, reference data

**Lookup Reference Table**:
A deterministic mapping table containing exact sensitive string pairs (such as project codenames, hostnames, and IP addresses) to be desensitized.
_Avoid_: Dictionary, redact list, secret map

**Blackout Masking**:
The process of drawing solid black rectangles over sensitive text detected inside images via local OCR.
_Avoid_: Image blurring, pixelation, image masking

**Clarification Request**:
A prompt generated by an agent when ambiguous or incomplete parameters are identified in desensitized documents, requiring human feedback via the Web UI.
_Avoid_: Agent question, HITL query, gap flag
